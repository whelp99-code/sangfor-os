- **치명적**: **`/deals`·`/opportunities` 전체가 500**(영업 메인 작업공간 다운, nuqs 어댑터 누락). 단순 1곳 수정으로 복구 가능 → **최우선**.

- **제품 비전 갭**: "AI 자동화 포털"인데 **AI 기능 대부분이 설계만 되고 트리거가 사람 손**(메일 자동분류·딜 backfill·제안 초안 등 미가동), 에이전트 실행이 **휘발성(in-memory)**.
- **공통 근본원인**: ① 끊긴 연결(작업↔프로젝트, 재무↔딜 99% 미배정), ② 시드/실데이터 부재, ③ 결정·활동 로그 미수집 → AI 학습 신호 없음.

**권장 1순위 방향**: (1) `/deals` 500 즉시 수정 → (2) 가짜 수치/no-op 버튼 제거(신뢰 회복) → (3) 끊긴 연결 봉합(작업-프로젝트, 재무-딜 backfill UI) → (4) 이미 구현된 AI 파이프라인에 **자동 트리거 + 영속화** 부착.

## 2. 😣 불편한 점 (주제별)

**신뢰성 — "이 숫자 믿어도 되나?"**
- 빈 데이터 vs 진짜 0을 구분 못 함(6월 KPI 전부 0인데 안내문구 없음 → 오류로 오해).
- 같은 지표가 화면마다 다른 모수: 미수금(KPI=amount 285M vs 월결산=amount+vat 313M), 비용(딜별손익=전체 vs KPI=isPaid만).
- 가짜 수치가 실데이터처럼 섞여 전반적 신뢰 저하(B5).

**끊긴 연결 — "따로 노는 모듈들"**
- 재무 99.4% 매출·98.7% 비용이 **"미배정" 한 덩어리** → 딜별 손익 기능이 사실상 무력(backfill 1건뿐).
- 작업이 프로젝트(딜)와 분리 → 프로젝트별 작업 추적 불가, 담당자 지정 불가.
- 지식/메일/에이전트가 빈 상태로 시작(시드 미적재) → 첫 화면이 항상 빈손.

**조작성 — "클릭했는데 아무 일도"**
- no-op 버튼(필터/열설정/새 연락처), 자유텍스트 우선순위(enum과 불일치로 생성 실패), 상태변경마다 전체 페이지 reload(스크롤·포커스 손실).
- 미수금 페이지에 정렬/필터 미연결(비용엔 있는데) — 미수금 관리가 핵심인데 정작 못 거름.

## 6. 📊 수집이 필요한 데이터 (제품이 더 똑똑해지려면)

**결정·활동 로그(= AI 학습 신호, 현재 거의 0):**
- 딜 **단계 전환 이력/체류기간**(win rate·정체 탐지), 영업 **활동 로그**(메일/통화/미팅 — 세일즈 가짜수치의 근본원인).
- AI 제안 **승인/수정/반려 결정 로그**(`outcomeBreakdown` 스키마는 있으나 0건) → 자동반영 임계치·도메인 자율도 학습의 핵심.
- 메일 분류 **정답 라벨/교정쌍**, 검색 **쿼리→클릭 로그**(임베딩 재랭킹), 에이전트 **실행 trace 영속화**(성공/실패 도구체인).

**재무 정밀화:**
- proofType 표준화·카드/현금 구분(VAT 모수), 계좌 시재 잔액, 딜-원가 매핑(매입처별 어느 딜), 부가세 신고구분(영세율/면세/카드매출).

**성과 연결(가장 가치 큼):**
- 메일 근거(evidence)→최종 딜 성사 연결, PoC 성공률→수주 전환, 견적 vs 실집행(마진 예측).
